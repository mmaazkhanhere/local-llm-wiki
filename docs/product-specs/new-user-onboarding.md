# New User Onboarding

## Goal

Get a user from first launch to first generated wiki output with as little confusion as possible.

## Required Flow

1. explain what the app does
2. state clearly that raw notes will not be modified
3. select an existing Obsidian vault
4. preview folders the app will create or manage
5. enter the provider key
6. test provider connectivity
7. start indexing
8. show first generated outputs and any errors

## Copy Requirements

The UI must tell the user:

- vault data stays local
- some extracted source content may be sent to the configured provider
- generated files appear in app-owned wiki folders
- raw files remain untouched
