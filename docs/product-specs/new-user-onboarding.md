# New User Onboarding

## Goal

Get a non-technical user from first launch to first generated summary with minimal decisions.

## Flow

1. Open app.
2. See clear explanation that raw notes will not be modified.
3. Select existing Obsidian vault.
4. Review folders that will be created.
5. Enter Groq API key.
6. Test connection.
7. Start indexing.
8. See first generated outputs and any failures.

## Copy Requirements

Must tell the user:

- notes stay on the machine
- some extracted content may be sent to the configured provider
- generated files will appear inside `LLM Wiki/`
- raw notes will not be modified
