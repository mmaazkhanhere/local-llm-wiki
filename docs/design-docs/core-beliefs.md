# Core Beliefs

## Product Beliefs

1. Raw notes are the source of truth and must remain untouched.
2. Generated pages are valuable because they improve navigation and recall, not because they replace evidence.
3. Local-first is a real product constraint, not a marketing phrase.
4. The app should compile knowledge into persistent markdown instead of re-synthesizing everything from scratch on every question.
5. User trust depends more on provenance, visibility, and safe writes than on model cleverness.

## Engineering Beliefs

- Safety beats novelty in the MVP.
- Deterministic file ownership matters more than clever automation.
- A small number of explicit page types is better than unconstrained generation.
- Retrieval should stay simple until quality data proves it is insufficient.
- Generated output must remain understandable without the app.
